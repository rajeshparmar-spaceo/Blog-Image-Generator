# Image Guidelines

A comprehensive guide for creating consistent, high-quality blog images for Monocubed.

---

## Quick Reference

| Specification | Value |
|---------------|-------|
| **Dimensions** | 1000px x 500px |
| **Aspect Ratio** | 2:1 (Landscape) |
| **Max File Size** | 100 KB |
| **Preferred Format** | WebP |
| **Font Family** | Poppins |
| **Logo Style** | White semi-circle badge, bottom-right corner |

---

## 1. Technical Requirements

### File Formats

| Format | Use Case |
|--------|----------|
| **WebP** | Preferred for production (best quality/performance balance) |
| **PNG** | When transparency is required (logos, overlays, UI elements) |
| **JPEG** | Realistic photographic backgrounds with no transparency |

### Dimensions & Size

- **Featured Image:** 1000px (W) x 500px (H)
- **Aspect Ratio:** 2:1 (Landscape) — must be followed strictly
- **File Size:** Keep under 100 KB
- **Tools:** TinyPNG, Squoosh, ImageOptim for compression
- **Quality:** No pixelation, blur, or artifacts

## 2. Color Palette

### Brand Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Primary Blue** | `#2B7DE9` | Icons, accent elements, decorative circles, logo cube |
| **Light Blue** | `#4DA3FF` | Icon outlines, lighter accent elements |
| **Very Light Blue** | `#B3D4FF` | Wireframe illustrations, faint UI outlines |
| **White** | `#FFFFFF` | Icon circle backgrounds, logo badge |
| **Light Gray** | `#F5F7FA` | Subtle background tint |

### Text Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Near-Black** | `#1A1A1A` | Headlines (primary text) |
| **Dark Gray** | `#4A4A4A` | Subtitles |
| **Medium Gray** | `#666666` | Body text (rare) |

### Background Treatment

The Monocubed style uses **realistic office photography** as the background, with the left text area naturally lighter to ensure text readability.

| Property | Specification |
|----------|---------------|
| **Background type** | Realistic office/workspace photography |
| **Left side (~35-40%)** | Lighter/washed out for text — natural bright area of photo, or fading to white/light gray on left edge |
| **Right side (~60-65%)** | Person at desk with laptop/monitor, sharper detail |
| **Lighting** | Soft, natural, bright office lighting |
| **Depth** | Background behind person slightly blurred/bokeh |
| **Overall tone** | Clean, bright, professional, modern office |

## 3. Composition Guidelines

### Primary Layout: Left Text / Right Visual (Used in 100% of reference images)

| Zone | Position | Width | Content |
|------|----------|-------|---------|
| **Text Zone** | Left side | ~35-40% of canvas | Headline + subtitle, vertically centered or top-aligned |
| **Visual Zone** | Right side | ~60-65% of canvas | Person at desk/workspace with laptop/monitor |
| **Icon Zone** | Upper-center (between text and person) | Overlapping both zones | Floating topic-relevant icons/illustrations |
| **Logo Zone** | Bottom-right corner | ~170px wide | Monocubed logo in white badge |

### Text Zone Specifications

| Property | Specification |
|----------|---------------|
| **Alignment** | Left-aligned |
| **Horizontal position** | Left edge with ~40-60px left padding |
| **Vertical position** | Vertically centered or slightly top-heavy (top 30-70% of canvas) |
| **Max width** | ~35-40% of canvas width (~350-400px) |
| **Headline lines** | 2-4 lines (wraps naturally due to narrow width) |
| **Subtitle lines** | 1-3 lines |
| **Gap between headline and subtitle** | ~12-20px |

### Layout Variants

| Variant | Usage | Description |
|---------|-------|-------------|
| **Standard (Type A)** (~82%) | Single person at desk | Text left, single person at desk right, optional floating icons between |
| **Multi-person (Type B)** (~9%) | Team/meeting scenes | Text left, multiple people in office/meeting right |
| **Step List (Type C)** (~9%) | How-to/tutorial guides | Text + numbered list on left, person in circular frame center, floating screenshots right |

### Core Design Values

- Clean
- Modern
- Functional
- Purpose-driven (every element supports the blog topic)

## 4. Human Photography Rules

### Subject

| Property | Specification |
|----------|---------------|
| **Subject** | Professional person at a desk/workspace |
| **Gender** | Both men and women — predominantly men (~73%), women (~18%), mixed/group (~9%) |
| **Quality** | High-quality, realistic photography only |
| **Avoid** | Illustrations, vector characters, cartoon-style humans |

### Positioning & Direction

| Property | Specification |
|----------|---------------|
| **Position** | Right side of canvas (~60-65% area) |
| **Facing direction** | Facing LEFT toward the text area (most common) or slightly toward camera |
| **Expression** | Focused, professional, sometimes smiling |
| **Posture** | Seated at desk (most common), sometimes standing (presentation scenes) |
| **Attire** | Professional/business casual — button-down shirts, blazers, polos |
| **Cropping** | Upper body to full seated figure — person should NOT touch canvas edges |

### Equipment & Props

| Property | Specification |
|----------|---------------|
| **Laptop** | Present in ~100% of images — always open, showing topic-relevant content |
| **External monitor** | Present in ~27% of images — showing code or secondary content |
| **Screen content** | MUST be relevant to article topic (ecommerce sites, dashboards, code, analytics) |
| **Desk items** | Coffee cup, phone, notebook occasionally visible — subtle, not distracting |

### Screen Content by Topic

| Topic | Screen Content |
|-------|---------------|
| **Ecommerce** | Product listing pages, online store fronts, shopping carts |
| **Development** | Code editors with syntax highlighting, IDE views |
| **Analytics/Cost** | Dashboard with charts, graphs, metrics |
| **Payment** | Payment interface, checkout flow |
| **Insurance** | Insurance portal dashboard with stats |
| **WordPress** | WordPress block editor, hosting panel |
| **Outsourcing** | Ecommerce site + video call with team on second screen |

### Photo Background

- Soft, blurred, or minimal office environment
- Free from distractions
- Bright, natural lighting — no harsh shadows or dark/moody lighting
- Easy to overlay text on the left side (lighter/washed-out left area)

## 5. Typography

### Font Specifications

| Element | Size | Weight | Color |
|---------|------|--------|-------|
| **Headline** | 30-34px | 700 (Bold) | `#1A1A1A` |
| **Subtitle** | 18-22px | 400-500 (Regular to Medium) | `#4A4A4A` |
| **Body / List items** | 14-16px | 400 (Regular) | `#1A1A1A` |

**Font Family:** Poppins

### Text Rules

- Keep text **minimal and impactful**
- Maximum: **5-10 words** per line
- Maintain strong contrast — dark text on light/washed-out left side
- **Headline and subtitle are always separate** — headline is large + bold, subtitle is smaller + lighter
- **Avoid:** Script fonts, decorative fonts, condensed or overly stylized typefaces
- Text is **always left-aligned**, positioned in the left ~35-40% of canvas

### Text Hierarchy Pattern (100% consistent)

```
[Headline — Bold 700, 30-34px, near-black #1A1A1A, 2-4 lines]

[Subtitle — Regular/Medium 400-500, 18-22px, dark gray #4A4A4A, 1-3 lines]
```

## 6. Floating Icons & Illustrations

Present in ~82% of reference images. These are **topic-relevant decorative elements** floating between the text zone and person.

### Icon Styles

| Style | Usage | Description |
|-------|-------|-------------|
| **Blue circle icons** (~55%) | Most common | Topic icon inside white circle with blue (`#2B7DE9`) border. Size ~50-70px. |
| **Hexagonal icon grid** (~11%) | Tool comparisons, payment gateways | Company/service logos inside gray hexagonal outlines, honeycomb cluster |
| **Blue wireframe illustrations** (~11%) | UI/platform topics | Light blue outlined browser windows, checklists, UI mockups. ~100-150px. |
| **Floating screenshots** (~11%) | Tutorials, how-to guides | Real app/website screenshots floating with subtle shadow |
| **Dark circle icons** (~11%) | Variant | Dark/black fill circle with white icon inside |

### Icon Positioning

| Property | Specification |
|----------|---------------|
| **Primary position** | Upper-center of canvas, between text zone and person |
| **Curved arrow** | ~36% of images include a small curved arrow from text area toward the icon |
| **Arrow style** | Thin black curved line with small arrowhead, hand-drawn feel |
| **Spacing** | Icons float above background with subtle shadows — overlaid, not embedded |

### Icon Cluster Arrangements

| Arrangement | Description |
|-------------|-------------|
| **Single icon** | One prominent circle icon, center-upper area |
| **Dual icons** | Two circle icons stacked or side by side |
| **Arc cluster** | 4-6 icons in a curved arc above the person |
| **Hexagonal grid** | 5-6 icons in a honeycomb pattern |
| **Scattered** | 2-3 floating elements at different depths |

### Icon Content by Topic

| Topic | Typical Icons |
|-------|--------------|
| **Ecommerce** | Shopping cart, store/shop, product catalog, browser window |
| **Payment** | Dollar sign, credit card, payment gateway logos |
| **Development** | Code brackets, gears, document/clipboard, API flowchart |
| **Insurance** | Shield, scales/legal, handshake, calendar, user profile |
| **Website/WordPress** | Browser window, wireframe, checklist, settings gear |
| **Cost/Pricing** | Coin stack, dollar sign circle, gear with magnifier |
| **Outsourcing** | Globe with ecommerce, team/people, video call |

## 8. Brand Logo

### Logo File

| Property | Value |
|----------|-------|
| **File** | `Monocubed-Logo-blue-full@2x.png` |
| **Contents** | Blue 3D cube icon + "Monocubed" dark text |

### Logo Badge Style

The Monocubed logo is displayed inside a **white semi-circle/oval badge** that clips at the bottom-right corner of the canvas.

| Property | Value |
|----------|-------|
| **Position** | ALWAYS bottom-right corner |
| **Badge shape** | White semi-circle / partial oval |
| **Badge behavior** | Clips at bottom-right edge — extends slightly beyond canvas |
| **Badge size** | ~160-180px wide, ~55-65px tall (visible portion) |
| **Logo inside badge** | Blue cube icon (~25-30px) + "Monocubed" text |
| **Badge background** | White (`#FFFFFF`) with very subtle shadow on curved edge |
| **Margin** | 0px from right edge, 0px from bottom edge |

### Placement Rules

- Logo badge is present in **100% of reference images**
- Always bottom-right — never any other position
- The white badge creates a clean area for the logo even over busy photo backgrounds
- Keep subtle — badge should not dominate the composition

## 9. Brand Consistency

Maintain uniformity across all blog images:

- **Color tones consistent** — Primary Blue `#2B7DE9` for all accents
- **Lighting style consistent** — bright, natural office lighting
- **Typography placement consistent** — left-aligned, Poppins, same weight hierarchy
- **Icon style consistent** — blue outlined circles or topic-appropriate variant
- **Logo placement consistent** — white badge, bottom-right, same size
- **Accessibility:** WCAG 2.1 AA compliant contrast ratios (4.5:1 minimum)

---

## 10. Common Mistakes to Avoid

| Don't | Do Instead |
|-------|------------|
| Cluttered layouts with too many elements | Keep designs clean and focused |
| Low-contrast or unreadable text | Ensure 4.5:1 minimum contrast |
| Over-designed gradients or busy patterns | Use subtle, purposeful design |
| Generic stock photos without relevance | Select photos supporting article topic |
| Inconsistent icon, font, or color usage | Follow established style guide |
| Misaligned logo or oversized branding | Use logo badge at standard size, bottom-right |
| Illustrations or cartoon-style people | Use realistic photography only |
| Person facing right (away from text) | Person should face LEFT toward text |
| Screen showing irrelevant content | Laptop/monitor must show topic-relevant content |
| Placing icons over the person's face | Icons go between text and person, upper area |
| Equal left-right width split | Text zone ~35-40%, visual zone ~60-65% |
| Using different fonts or weights | Stick to Poppins — Bold headlines, Regular subtitles |

---

## 11. AI Image Generation Tips

When using AI tools to generate images:

- Use realistic photography for human backgrounds — no illustrations
- Person seated at desk with laptop, facing LEFT
- Office environment: bright, modern, slightly blurred background
- Laptop screen should show topic-relevant content (ecommerce site, dashboard, code, etc.)
- Leave the left ~35-40% of the image lighter/less busy for text overlay
- Add floating blue-circle icons between text area and person for topic relevance
- Follow correct placement: text on left, human on right, logo badge bottom-right
