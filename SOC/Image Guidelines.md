## 1. Technical Requirements

### File Formats

| Format | Use Case |
|--------|----------|
| **WebP** | Preferred for production (best quality/performance balance) |
| **PNG** | When transparency is required (logos, overlays, UI elements) |
| **JPEG** | Realistic photographic backgrounds with no transparency |

### Dimensions & Size

- **Featured Image:** 1416px (W) x 748px (H)
- **Aspect Ratio:** 16:9 (Landscape) - must be followed strictly
- **File Size:** Keep under 100 KB
- **Tools:** TinyPNG, Squoosh, ImageOptim for compression
- **Quality:** No pixelation, blur, or artifacts

## 2. Composition Guidelines

### Core Design Values

- Clean
- Modern
- Functional
- Healthcare-focused and trustworthy
- Purpose-driven (every element supports the blog topic)

### Primary Layout: Two-Zone Split (Used in 100% of reference images)

| Zone | Width | Content |
|------|-------|---------|
| **Left Text Zone** | 25–35% of canvas | Headline, subtitle, bulleted feature list with icons, or company logo badges |
| **Right Visual Zone** | 65–75% of canvas | Photography with healthcare UI dashboards and app mockups on screens |

- A soft gradient fade (white `#FFFFFF` to light gray `#F5F7FA`) transitions between the text zone and photo — no hard edge
- The left zone uses a semi-transparent light wash to ensure headline readability over any photo bleed

### Layout Variants

| Variant | Trigger / Use Case | Description |
|---------|-------------------|-------------|
| **Type A – "Healthcare Guide / Development"** | Telehealth, EHR, patient monitoring, healthcare software | Headline left (25–35%); professional at multi-screen workstation right (65–75%) showing healthcare dashboards/patient portals on screen; circular icon badge floating above laptop with hand-drawn arrow; subtitle below headline |
| **Type B – "Top Companies / Listicle"** | Top development companies, provider lists | Headline left (30%); healthcare team at workstation right (70%) with EHR/dashboard on monitors; company logo badges in 2×3 grid below headline in left zone |
| **Type C – "App Development / Outsourcing"** | Mobile apps, outsource development | Headline left (30%); mobile phone mockup showing app UI + code editor browser window in center-right; world map illustration with location pins in lower-left (for outsourcing); 3D elements (money, coins, shields) for cost articles |
| **Type D – "Feature / Benefit Breakdown"** | Software features, implementation guides | Headline left (30–35%); professional at desk with laptop right (65–70%); bulleted feature list below headline with small circular icons (e.g., "Features", "Benefits", "Cost", "Process"); clean photo with no complex overlays |
| **Type E – "Canadian-Specific Content"** | Articles targeting Canadian healthcare market | Same as Type A but with Canadian flag visible in background; emphasis on Canadian compliance and regulations; stronger use of red (#E74C3C) accents |

### Text Placement Rules

| Property | Specification |
|----------|---------------|
| Headline alignment | Always left-aligned in the left zone |
| Headline vertical position | Upper-center of left zone |
| Headline stacking | Multi-line with 1–2 words per line for maximum impact |
| Subtitle position | Directly below headline, lighter weight (500), smaller size (24px) |
| Subtitle style | Regular case, softer color (`#555555`), often describes "Features, Cost, and Step-by-Step Guide" format |
| Text max-width | 25–35% of canvas width |
| Left padding | 50px |
| Top padding | 50px |

### Bulleted Feature Lists (Distinctive SOC Element)

A unique SOC pattern — used in Type D layouts below the headline:

| Property | Specification |
|----------|---------------|
| **Placement** | Below headline and subtitle in left text zone |
| **Format** | 3 vertical bullet items with circular icons on the left |
| **Icon style** | Dark circular badges (~35–40px) with thin white outlined icons inside |
| **Labels** | Short keywords: "Features", "Benefits", "Cost", "Process", "Implementation Guide" |
| **Spacing** | 12–15px vertical gap between items |
| **Text** | Poppins, 16px, regular weight, dark gray |

### Logo Placement Rules

| Property | Specification |
|----------|---------------|
| **Primary position** | Bottom-right corner with white semi-circle/arc badge background |
| **Alternative position** | Top-right corner with white semi-circle/arc badge background |
| **Badge style** | White semi-circle/arc cutout — this is the DOMINANT style for SOC (used in ~90% of images) |
| **Margin** | 15px from edges |
| **Size** | Height 44–50px, width proportional |
| **Branding** | Red maple leaf icon + red 'O' circle (Canadian identity) |

### Photography Composition Rules

| Property | Specification |
|----------|---------------|
| **Subject** | Healthcare professional(s) or developer(s) at desk with laptop AND monitor(s) |
| **Multi-screen** | Multi-monitor setups are very common — laptop + external monitor(s) showing different content |
| **Camera angle** | Side profile or 3/4 view |
| **Face direction** | Person facing LEFT (toward the text zone) |
| **Positioning** | Person placed in right 50–65% of canvas |
| **Expression** | Neutral, focused, professional |
| **Environment** | Clean, bright, modern office/workspace with natural light |
| **Depth of field** | Shallow — softly blurred background |
| **Screen content** | Healthcare dashboards, patient portals, EHR systems, vitals monitoring, video calls with patients, or code editors |

#### Category-Specific Photography

| Category | Subject Details |
|----------|----------------|
| **Telehealth** | Professional at desk with monitor showing patient video call + vitals dashboard; HIPAA security shields floating |
| **EHR** | Healthcare team (doctor + developers) at workstation with patient record system on screens |
| **App Development** | Mobile phone mockup showing app UI + code editor browser window behind; professional or 3D elements |
| **Remote Monitoring** | Person at laptop with patient monitoring dashboard showing heart rate, vitals, video feed of patient |
| **MVP / Team** | Multi-person office scene — person at monitors + team member at whiteboard with Kanban board |
| **Mental Health** | Professional at laptop with EHR patient profile; Canadian flag in background for Canadian-market articles |

### Floating UI Overlay & Accent Element Rules

| Element | Placement | Style |
|---------|-----------|-------|
| **Healthcare dashboards** | Visible on monitor/laptop screens in the photo | Realistic UI with patient data, vitals charts, appointment lists |
| **Circular icon badges** | Floating above laptop, connected by hand-drawn curved arrow | Circle (50–60px) with thin outlined healthcare icon inside (heart, patient, lock), hand-drawn arrow pointing from icon to screen |
| **Hand-drawn arrows** | Connecting floating icon badge to laptop/screen | Thin, curved, dark stroke, casual hand-drawn feel |
| **Company logo badges** | 2×3 grid below headline (Type B) | White rounded-rectangle cards (~130×35px) with subtle border/shadow, 10px gap |
| **Mobile phone mockups** | Center-left area, overlapping text-photo boundary | Realistic phone frame showing app UI (doctor listings, booking screens) |
| **Code editor windows** | Behind phone/person as background element | Browser window with code (JavaScript), slightly enlarged and faded |
| **3D accent elements** | Near phone mockup or bottom area (Type C) | Money stacks, coins, security shields with dollar signs — used sparingly for cost articles |
| **World map illustration** | Lower-left zone (outsourcing articles only) | Light blue/gray world map with location pins and dotted connection lines |
| **Security shields** | Floating near monitors (HIPAA/compliance articles) | Blue shield icons with lock/check symbols |
| **Canadian flag** | Background element in photo (Canada-specific articles) | Real Canadian flag visible in office environment |

### Spacing & Proportion Rules

- Use generous white space in the text zone — no clutter
- Follow the Rule of Thirds for balanced composition
- Subject should not touch canvas edges — maintain 15–20px breathing room
- Multi-monitor setups: primary screen shows relevant dashboard, secondary screen can show code or additional data
- Floating elements should not obscure headline text
- Bulleted feature lists: align vertically, maintain consistent icon sizing
- Maintain 15–20px gap between headline and subtitle, and 20–25px between subtitle and feature list

## 3. Human Photography Rules

When using realistic human images:

### Requirements

- Use **high-quality, realistic** human photography only
- **Avoid:** Illustrations, vector characters, cartoon-style humans

### Positioning

- Person positioned on the **right side**
- Face direction: **toward the left**
- Expression: Neutral, focused, professional
- Posture: Sitting preferred

### Background

- Soft, blurred, or minimal
- Free from distractions
- Easy to overlay text on the left side

## 4. Typography

### Font Specifications

| Element | Size | Weight |
|---------|------|--------|
| **Heading** | 32-34px | 600 (Medium) |
| **Sub-heading** | 24px | 500 (Medium) |
| **Body** | 16px | 400 (Regular) |

**Font Family:** Poppins

### Text Rules

- Keep text **minimal and impactful**
- Maximum: **5-10 words only**
- Maintain strong contrast:
  - Dark text on light backgrounds
  - Light text on dark/gradient backgrounds
- **Avoid:** Script fonts, decorative fonts, condensed or overly stylized typefaces

---

## 5. Visual Elements

### Icons

- Style: **Outline only**
- Stroke: **2px thickness**
- Consistent line weight across all icons
- Support the message, don't dominate it

### Graphics & Illustrations

Use as secondary or foreground elements:

- **3D isometric elements** - Modern, tech-forward look
- **Flat design with depth** - Subtle shadows, soft highlights
- **Abstract shapes** - Circles, rounded rectangles, organic blobs
- **Tech symbols** - Laptops, mobile devices, dashboards, AI icons, documents, charts, workflows

**Warning:** Avoid mixing too many styles in one image.

---

## 6. Brand Logo

### Placement

- **Preferred:** Right-bottom corner
- **Alternative:** Center-bottom (if composition allows)

### Size

- **Height:** 44px (fixed)
- **Width:** Scaled proportionally

### Guidelines

- Keep subtle - never overpower the main visual
- Maintain clear padding from edges
- Use only the official brand logo

## 7. Consistency Checklist

Maintain uniformity across all blog images:

- [ ] Color tones consistent
- [ ] Lighting style consistent
- [ ] Typography placement consistent
- [ ] Icon style consistent

---

## 8. Accessibility (WCAG 2.1 AA)

- **Minimum contrast ratio:** 4.5:1 for text
- Text must be readable at smaller sizes
- No critical information conveyed by color alone

---

## 9. Common Mistakes to Avoid

| Don't | Do Instead |
|-------|------------|
| Cluttered layouts with too many elements | Keep designs clean and focused |
| Low-contrast or unreadable text | Ensure 4.5:1 minimum contrast |
| Over-designed gradients or busy patterns | Use subtle, purposeful design |
| Generic stock photos without relevance | Select images that support content |
| Inconsistent icon, font, or color usage | Follow established style guide |
| Misaligned logo or oversized branding | Place logo subtly in corner at 44px |

---

## 10. AI Image Generation Tips

When using AI tools to generate images:

- Use realistic photography for human backgrounds
- Create clean tech visuals for overlays
- Avoid random props or unrelated objects
- Follow correct placement: human on right, text space on left
