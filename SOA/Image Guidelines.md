## 1. Technical Requirements

### File Formats

| Format | Use Case |
|--------|----------|
| **WebP** | Preferred for production (best quality/performance balance) |
| **PNG** | When transparency is required (logos, overlays, UI elements) |
| **JPEG** | Realistic photographic backgrounds with no transparency |

### Dimensions & Size

- **Featured Image:** 708px (W) x 374px (H)
- **Aspect Ratio:** ~1.89:1 (Landscape) — must be followed strictly
- **File Size:** Keep under 100 KB
- **Tools:** TinyPNG, Squoosh, ImageOptim for compression
- **Quality:** No pixelation, blur, or artifacts

## 3. Composition Guidelines

### Primary Layout: Two-Zone Split (Used in 100% of reference images)

| Zone | Width | Content |
|------|-------|---------|
| **Left Text Zone** | 25-30% of canvas | Headline (stacked 1-2 words/line), subtitle, optional small illustrated icons |
| **Right Visual Zone** | 70-75% of canvas | Photography with topic-relevant screen content, optional floating UI overlays |

### Text Zone Specifications

| Property | Specification |
|----------|---------------|
| **Alignment** | Left-aligned |
| **Left padding** | ~50px |
| **Top padding** | ~50px |
| **Vertical position** | Vertically centered in left zone |
| **Max width** | 25-30% of canvas width |
| **Headline stacking** | Multi-line with **1-2 words per line** for maximum visual impact — creates tall, narrow headline blocks |
| **Headline lines** | 3-5 lines (due to narrow stacking) |
| **Subtitle position** | Directly below headline |
| **Gap** | ~10-16px between headline and subtitle |

### Layout Variants

| Variant | Usage | Description |
|---------|-------|-------------|
| **Type A -- Guide / Technical Article** (~57%) | AI/ML guides, development articles, telemedicine | Headline left (25-30%); professional at workstation right (70-75%); screen shows relevant dashboard/UI; optional small illustrated icons in lower-left below headline; optional floating overlay cards near screen |
| **Type B -- Top Companies / Listicle** (~14%) | Consulting companies, service provider lists | Headline left (28-30%); group photo of professionals in meeting/office right (70-72%); clean photo with NO overlays; subtitle below headline |
| **Type C -- Product / AI Feature Showcase** (~14%) | Ecommerce AI, chatbot, product-focused articles | Headline left (25-28%); person at laptop right; floating product cards + chatbot/AI conversation windows layered in center overlapping both zones; soft mint/teal gradient background on left |
| **Type D -- Mobile / App Focus** (~14%) | Symptom checkers, mobile apps, phone-based interfaces | Headline left (28-30%); person holding smartphone right; app UI visible on phone screen; holographic/translucent floating UI elements above the device |

## 4. Typography

### Font Specifications

| Element | Size | Weight | Color |
|---------|------|--------|-------|
| **Headline** | 28-32px | 600-700 (Semi-Bold to Bold) | `#1A1A1A` |
| **Subtitle** | 16-20px | 400-500 (Regular to Medium) | `#555555` |
| **Body** | 14-16px | 400 (Regular) | `#666666` |

**Font Family:** Poppins

### Headline Style

- **Stacking:** 1-2 words per line — creates a tall, narrow, impactful text block
- **Alignment:** Always left-aligned
- **Case:** Title case or sentence case (not all caps)
- Headline occupies the full height of the left zone due to stacking

### Subtitle Style

- Noticeably lighter than headline — both in weight and color
- Positioned directly below headline
- Sometimes appears slightly italic (varies)
- Regular case, softer gray color (`#555555`)

### Text Rules

- Keep text **minimal and impactful**
- Maximum: **5-10 words** for headline content
- Maintain strong contrast — dark text on light/washed-out left side
- **Avoid:** Script fonts, decorative fonts, condensed or overly stylized typefaces

## 5. Human Photography Rules

### Subject Types by Category

| Category | Subject | Attire | Screen Content |
|----------|---------|--------|----------------|
| **Healthcare / Telemedicine** | Doctor at desk with monitor | White coat | Medical dashboards, AI neural networks, patient video calls, vitals, X-rays |
| **AI / ML** | Developer/professional at laptop | Business casual | AI dashboards, MLOps tools, neural network visualizations, data runs |
| **Consulting / Listicle** | Group of professionals in meeting room | Business casual / blazers | Charts, graphs, presentations on laptops |
| **Ecommerce** | Person at laptop | Casual / business casual | Product interfaces visible via floating overlay cards |
| **Mobile Apps** | Person holding smartphone | Casual / professional | App UI visible on phone screen |

### Positioning & Direction

| Property | Specification |
|----------|---------------|
| **Position** | Right side of canvas (~70-75% area) |
| **Facing direction** | Facing LEFT toward text zone (~71% of images). Exception: Type D mobile variant may face RIGHT to show phone screen. |
| **Camera angle** | Side profile or 3/4 view |
| **Expression** | Neutral, focused, professional — sometimes smiling (group photos) |
| **Posture** | Seated at desk (most common), standing (rare), holding device (Type D) |

### Gender Distribution (observed)

| Gender | Frequency |
|--------|-----------|
| **Male (individual)** | ~57% |
| **Female (individual)** | ~29% |
| **Mixed group** | ~14% |

### Photo Background

- Clean, bright, modern workspace with natural light
- Shallow depth of field — softly blurred background
- Free from visual clutter
- Left portion naturally lighter for text overlay

### Screen Content (Critical SOA Element)

The content on laptop/monitor screens is a **primary visual element** — it communicates the article topic directly. Screens show detailed, realistic, topic-relevant UI.

| Topic | Screen Content Details |
|-------|----------------------|
| **Telemedicine / ML** | Complex multi-panel medical dashboards: brain neural network viz, patient vitals (BP, temp), video calls with patients, X-rays, EHR data, urgency indicators |
| **Agentic AI** | Dark-themed dashboard with patient info, urgency scores, AI insights, autonomous actions, symptom analysis |
| **Generative AI** | Video call with patient + clinical notes overlay (Symptoms, Assessment, Plan) |
| **MLOps** | Data runs table, model performance charts, waveform visualizations |
| **Ecommerce** | Screen content shown via floating overlay cards (not directly on screen) |
| **Mobile** | Symptom checker app: symptom list, urgency level, confidence %, recommendation |

## 6. Floating UI Overlays

Floating UI overlays are a **distinctive SOA visual storytelling element**. Present in ~57% of reference images. They convey the article topic by showing actual product interfaces.

### Overlay Types

| Element | When Used | Placement | Style |
|---------|-----------|-----------|-------|
| **App screenshots / UI mockup cards** | Type A guide articles | Overlaid on or near the monitor screen, slightly angled or straight | White/dark cards with realistic UI content, subtle drop shadow |
| **Clinical notes card** | Healthcare articles | Overlaid on laptop screen or floating beside it | White card with structured sections: Symptoms, Assessment, Plan |
| **Chat / conversation window** | Ecommerce AI, chatbot articles (Type C) | Center of canvas, overlapping text-photo boundary | Rounded white card with chat bubbles, light blue (`#3498DB`) accent border at top |
| **Product cards** | Ecommerce articles (Type C) | Floating above or beside laptop/monitor | White card with product image, price, star rating, action buttons |
| **Holographic / translucent UI** | Mobile app articles (Type D) | Floating above phone/device | Semi-transparent panels with body scan silhouette, risk indicators (color-coded: red HIGH RISK, orange MODERATE, green LOW), chat bubbles, lock/security icon |
| **Circular tool/brand icon** | Tool list articles | Floating above laptop with hand-drawn arrow | Circular badge (~50-60px) with tool logo, connected by hand-drawn curved arrow |

### Chat Window Details (Type C)

| Property | Specification |
|----------|---------------|
| **Header** | "AI Assistant" label at top |
| **Border** | Light blue (`#3498DB`) top border accent |
| **User message** | Avatar + message text in white/light bubble |
| **Bot response** | Bot icon + response text |
| **Card shape** | Rounded rectangle with subtle shadow |
| **Position** | Center of canvas, overlapping both text and visual zones |

### Holographic UI Details (Type D)

| Property | Specification |
|----------|---------------|
| **Opacity** | Semi-transparent (~60-70% opacity) |
| **Content** | Body scan silhouette, risk level indicators (color-coded bars), chat bubbles, security icons |
| **Risk colors** | Red (#E74C3C) HIGH RISK, Orange (#F39C12) MODERATE RISK, Green (#2ECC71) LOW RISK |
| **Position** | Floating above/behind the smartphone |
| **Effect** | Holographic/futuristic feel — suggests advanced technology |

## 9. Spacing & Proportion Rules

- Use generous white space in the text zone — no clutter
- Follow the Rule of Thirds for balanced composition
- Subject should not touch canvas edges — maintain 15-20px breathing room
- Floating UI cards: overlap the text-photo boundary to create visual cohesion
- Multiple floating cards should be layered with slight offsets (staggered, not aligned)
- Maintain 10-15px gap between stacked floating elements
- Headline stacking (1-2 words per line) creates natural breathing room

---

## 10. Brand Consistency

Maintain uniformity across all blog images:

- **Color tones consistent** — Primary Green `#2ECC71` for logo and icon accents
- **Lighting style consistent** — bright, natural office lighting
- **Typography consistent** — Poppins, stacked headline, lighter subtitle
- **Logo placement consistent** — top-right, no badge, every image
- **Screen content relevant** — always topic-appropriate dashboards/UIs
- **Accessibility:** WCAG 2.1 AA compliant contrast ratios (4.5:1 minimum for text)

---

## 11. Common Mistakes to Avoid

| Don't | Do Instead |
|-------|------------|
| Cluttered layouts with too many elements | Keep designs clean and focused |
| Low-contrast or unreadable text | Ensure 4.5:1 minimum contrast, use light overlay on left |
| Over-designed gradients or busy patterns | Use subtle mint gradient, barely perceptible |
| Generic stock photos without relevance | Select photos supporting article topic |
| Inconsistent icon, font, or color usage | Follow established style guide |
| Logo in wrong position | Always top-right, no badge |
| Illustrations or cartoon-style people | Use realistic photography only |
| Person facing right in Type A layouts | Person should face LEFT (except Type D mobile) |
| Irrelevant screen content | Monitor/laptop MUST show topic-relevant UI |
| Wide headline text blocks | Stack 1-2 words per line for narrow, impactful headlines |
| Floating overlays in listicle photos | Type B uses clean photos, no overlays |
| Using illustrated icons outside healthcare | Small illustrated icons only for healthcare/telemedicine |

---

## 12. AI Image Generation Tips

When using AI tools to generate images:

- Use realistic photography for human subjects — no illustrations
- Person seated at desk with laptop/monitor, facing LEFT
- Healthcare: doctor in white coat, medical dashboard on screen
- Tech/AI: developer at laptop with data/dashboard on screen
- Leave left ~25-30% lighter for text overlay
- Laptop/monitor screen should show detailed, realistic, topic-relevant UI
- Add floating UI overlay cards for product/ecommerce articles
- Follow correct placement: text on left, human on right, logo top-right
