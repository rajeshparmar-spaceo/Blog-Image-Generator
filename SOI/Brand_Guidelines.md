# Brand Guidelines

## 1. Color Palette & Backgrounds

### Primary Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Primary Green** | `#A3C423` | Core brand color, used for highlights, CTAs, and key accents |
| **Secondary Gray** | `#6C757D` | Supporting elements, secondary text, UI components |

### Background Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **White** | `#FFFFFF` | Primary background |
| **Light Gray** | `#F4F4F4`, `#F8F8F8` | Section dividers and subtle contrast areas |

### Status & Utility Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Danger / Error (Imperial Red)** | `#E23C3C` | Error states and destructive actions |
| **Warning / Alert (Gold Metallic)** | `#DBB031` | Warning messages and alerts |

### Text Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Primary Text (Eerie Dark)** | `#1A1818` | Headings and body text |

## 2. Typography & Text Treatment

### Font Family

- **Primary Typeface:** Poppins

### Font Sizes

| Element | Size |
|---------|------|
| Headings | 30px |
| Body Text | 16px (Regular) |

> Minor size adjustments are allowed for layout responsiveness.

### Font Weight Hierarchy

| Element | Weight |
|---------|--------|
| Headlines | Medium (600) |
| Subheadings | Medium (600) |
| Body Text | Regular (400) |

### Text Usage Guidelines

- Keep text overlays minimal (5–10 words max)
- Maintain high contrast (dark text on light backgrounds or vice versa)
- Avoid decorative or script fonts to preserve clarity and professionalism

## 3. Visual Style & Composition

### Iconography

- Outline-style icons only
- Stroke width: 2px
- Clean, consistent, and minimal

### Imagery Style

- Use realistic, high-quality images
- Avoid overly staged or generic stock visuals

### Illustration Style

- 3D isometric elements for a modern, tech-forward look
- Flat illustrations with subtle depth and shadows
- Abstract shapes: circles, rounded rectangles, organic forms
- Tech-focused elements: laptops, mobile devices, documents, AI symbols

### Layout & Composition Rules

#### Canvas Dimensions

| Format | Dimensions | Aspect Ratio | Usage |
|--------|------------|--------------|-------|
| **Standard (Blog Featured)** | 708px × 374px | ~1.89:1 | "Top Companies" lists, hiring/cost articles |
| **Wide (In-depth Guide)** | ~1400px × 748px | ~1.87:1 | Healthcare guides, development tutorials, technical deep-dives |

#### Primary Layout: Two-Zone Split (Used in ~85% of images)

| Zone | Width | Content |
|------|-------|---------|
| **Left Text Zone** | 30–40% of canvas | Headline, subtitle, process icons, or company logo badges |
| **Right Visual Zone** | 55–70% of canvas | Photography, UI mockups, overlaid elements |

- A soft gradient fade (white/light gray) transitions between the text zone and the photo zone — no hard edge
- The left zone has a semi-transparent light wash (`#FFFFFF` to `#F4F4F4` at ~85–95% opacity) to ensure text readability over any bleed-through imagery

#### Layout Variants

| Variant | Trigger / Use Case | Description |
|---------|-------------------|-------------|
| **Type A – "Top Companies" List** | Listicle posts ("Top X Companies") | Headline top-left; circular-cropped photo center; company logo badges stacked vertically on right with green dotted connector lines. OR headline top-left with company logo badges in a 2×3 grid below the headline |
| **Type B – "Guide / Tutorial"** | Technical guides, development articles | Headline left (30–40%); full-bleed photography right (60–70%) showing developer at workstation; floating UI mockup cards overlaid on photo; process flow icons (in circles with arrows) in lower-left below headline |
| **Type C – "Full-Width Infographic"** | Map/data-centric posts (outsourcing, nearshore) | Centered headline at top; full-width illustrated map or infographic below; location pins and connection arrows; logo bottom-right |
| **Type D – "Cost / Hire"** | Cost breakdowns, hiring guides | Headline left; person photo right (slightly blurred background); floating profile cards or pricing UI elements overlaid on photo; decorative 3D elements (coins, icons) as accents |

#### Text Placement Rules

| Property | Specification |
|----------|---------------|
| Headline alignment | Always left-aligned in the left zone |
| Headline vertical position | Vertically centered in left zone, biased toward upper third |
| Headline stacking | Multi-line with 1–3 words per line for maximum impact |
| Subtitle position | Directly below headline, lighter weight (400), smaller size |
| Text max-width | 30–40% of canvas width |
| Left padding | 60px (standard) to 80px (wide format) |
| Top padding | 60px (standard) to 80–100px (wide format) |

#### Logo Placement Rules

| Format | Position | Style |
|--------|----------|-------|
| **Standard (708×374)** | Bottom-right or top-right corner | Direct placement on image, no badge background |
| **Wide (~1400×748)** | Bottom-right corner | White semi-circle/arc badge background (distinctive brand element) |
| **Logo margin** | 20px from right edge, 20px from bottom edge | Consistent across all formats |

#### Photography Composition

- **Subject**: Professional person(s) working at computer/laptop in modern office
- **Camera angle**: Side profile or 3/4 view of person looking at screen(s)
- **Positioning**: Person placed in right 50–60% of canvas, facing inward toward the text zone
- **Environment**: Clean, bright, well-lit modern workspace with natural light
- **Screens visible**: Monitor(s) showing code, dashboards, or software interfaces
- **Background**: Softly blurred (shallow depth of field) office environment

#### Overlay & Decorative Elements

| Element | Placement | Style |
|---------|-----------|-------|
| **Floating UI cards** | Overlaid on photography zone, angled or straight | White cards with subtle shadow, showing app interfaces or dashboards |
| **Process flow icons** | Lower-left zone, below headline (Type B layouts) | Outlined icons inside circles (2px stroke), connected by dashed arrows, arranged in 2×3 or serpentine grid |
| **Company logo badges** | Right side stacked vertically (Type A) or 2×3 grid below headline | White rounded-rectangle cards with subtle border/shadow |
| **Connector lines** | Between logo badges or icons | Green (#A3C423) dotted/dashed lines with small dots at nodes |
| **Decorative circles** | Background of left text zone | Very light blue/gray (#E8EDF2) circles, large and partially cropped |
| **3D accent elements** | Near headline or floating in composition | Coins, illustrations, small isometric objects (used sparingly) |
| **Circular photo crop** | Center of canvas (Type A only) | Person photo inside circular frame with light border |

#### Spacing & Proportion Rules

- Use generous white space in the text zone — avoid visual clutter
- Apply the rule of thirds for balanced compositions
- Maintain minimum 20px margin between overlay elements and canvas edges
- Process flow icons: 50–60px diameter circles, 15–20px gap between them
- Company logo badges: ~140px × 40px each, 10–15px vertical gap between badges
- Floating UI cards should overlap the text-photo boundary to create visual cohesion

## 4. Imagery & Graphic Elements

### Core Visual Elements

- Document and notebook visuals (stacked papers, folders, files)
- AI and technology symbols (neural networks, brain icons, automation)
- Collaboration visuals (multiple users, shared workflows)
- Data visuals (charts, graphs, connected nodes)

### Photography Guidelines

- Minimal human presence—focus on tools, systems, and workflows
- Clean, organized workspace environments
- Screen mockups should clearly showcase the product interface when relevant

| Asset Type | Dimensions |
|------------|------------|

### File Size Optimization

- Target file size under 100KB
- Use compression tools while preserving visual quality

## 6. Brand Consistency & Accessibility

- Maintain a clean, modern, and functional design approach
- Ensure accessibility compliance
- **WCAG 2.1 AA contrast ratio:** minimum 4.5:1

## 7. Common Mistakes to Avoid

| Avoid |
|-------|
| Overcrowded or cluttered compositions |
| Low-contrast or hard-to-read text |
| Overuse of complex gradients or patterns |
| Generic stock imagery without context |
| Inconsistent visual styling across assets |
